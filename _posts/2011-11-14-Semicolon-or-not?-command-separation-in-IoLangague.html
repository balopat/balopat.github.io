In Io semicolons are optional. Or not? 




As usual in script languages, if you want to use multiple commands in one line, you have to separate them with semicolon, otherwise they are optional (i.e you can separate commands with the new line character). 


BUT 


When ";" is not optional



  1. When you are using the io command line interpreter: Multiline blocks just seem to be multiline blocks, use the semicolons!

  2. When you are using the command: io < yourfile.io  Use the semicolons even in multiline expressions!


The optional cases 



  1. When you are using the command: 
    io yourfile.io 

  2. When you are using the commands:  
    doFile, doRelativeFile 

  3. When you are using io as shell interpreter: 
    #!/usr/local/bin/io in the header
